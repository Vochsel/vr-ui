vr-ui

Created by Ben Skinner for the CSIRO as a summer vacation scholar.

Overview

VR-UI is a small Unity package which provides functionality to create simple UI elements and interact with existing Unity scenes and code.

The UI elements can be hooked into scripts via UnityEvent's or through extending the base component and overriding virtual functions. They can also be customized visually through custom component properties.

Demo

Installation

Simply drag the vr-ui.unitypackage into your unity project!

Usage

  1. Attach the corresponding VRUI_Controller script to your VR controller GameObject's.
    E.g Attach VRUI_ViveController component to one or both [CameraRig]/Controller's GameObject's.
  2. Place interactable prefabs (Assets/vr-ui/Prefabs/Interactables) in scene, scale to desired size.
  3. Hook into desired scripts with state based Unity Events.
  4. Enjoy!

Structure

  • Cursor - Assets/vr-ui/Prefabs/Resources/VRUI_Cursor
    All interactables function by hooking into Unity's trigger system. As such, anything you wish to interact with must have a collider and kinematic rigid body component attached.
    It also must be tagged with "VRUI-Cursor", this stops other colliders from affecting the input.
  • Buttons - Assets/vr-ui/Prefabs/Interactables/VRUI_Button
    Drag and drop this prefab to create a button. Change the button type to change it's behaviour between; Toggle, and Trigger.
  • Sliders - Assets/vr-ui/Prefabs/Interactables/VRUI_Slider
    Drag and drop this prefab to create a slider.
  • Menus - Assets/vr-ui/Prefabs/Menus/VRUI_Menu
    Drag and drop this prefab to create a menu. All Interactables placed as children of this will be captured and contained by the menu and a border produced around them.

Design

The design of VR-UI draws inspiration from traditional 2D user interface paradigms and combines years of UI development with this new technology and medium. Currently we have two forms of interaction designs available, an Interaction Surface, and an Interaction Volume.

The interaction functionality has been designed around Unity triggers and events, so theoretically they should work with any VR system and controller configuration as long as the controller collider has a kinematic RigidBody component attached.

The surfaces are more akin to traditional, flat, ui elements. They require a physical "press" to activate and to manipulate. Their activation threshold, design, and functionality can be customized through the Unity Editor, with support for more options planned.

The volumes differ in design, and require the user to physically enter a volume with their cursor and then press a configurable controller button to activate.
